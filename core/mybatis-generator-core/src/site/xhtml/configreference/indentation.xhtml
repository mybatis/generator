The <indentation> Element

The <indentation> element is used to define indentation specifications for generated code. By default, the generator produces code with the following indentation specifications:

Source Code Type Indentation Specifications
Java 4 Spaces per indent level
Kotlin 4 Spaces per indent level
XML 2 Spaces per indent level

This element allows you to change these defaults. This element is an optional child element of the <context> element, and the <generatorConfiguration> element. If specified at the <generatorConfiguration> level, the specifications will apply to all contexts unless overridden by specifying a configuration at the <context> level. It is also possible to supply an indentation configuration through the Java API for the generator. This is primarily supported for environments like Eclipse where indentation can be configured in an IDE. Any configuration specified with the Java API will NOT override settings in a configuration file, but will only be applied if no configurations are specified through XML configuration.

Important Note: Kotlin code is always formatted with four spaces per indent level as the Kotlin language specification suggests.

Optional Attributes

Attribute Description
javaIndentType This can be either "Tabs" or "Spaces". "Spaces" is the default.
javaIndentAmount This should be a positive integer. If "javaIndentType" is spaces, the default is "4". If "javaIndentType" is "Tabs", the default is "1".
xmlIndentType This can be either "Tabs" or "Spaces". "Spaces" is the default.
xmlIndentAmount This should be a positive integer. If "xmlIndentType" is spaces, the default is "2". If "xmlIndentType" is "Tabs", the default is "1".

Child Elements

None.

Example

This element specifies that we want to use tabs for spacing with both Java and XML:

<indentation javaIndentType="Tabs" xmlIndentType="Tabs" />

This element specifies that we want to use two spaces for both Java and XML:

<indentation javaIndentAmount="2" />
