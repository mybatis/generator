The <javaMergeConfiguration> Element

The <javaMergeConfiguration> element is used to define the charcteristics of the Java merger. Please see the Java Merge Reference page for detailed information about how the Java merger works and the different options for configuring it. The default Java merger has the following characteristics:

Attribute Value
Lexical Preserving false
Import Sort Strategy Eclipse
Merge Strategy Merge_Into_Existing

This element allows you to change these defaults. This element is an optional child element of the <generatorConfiguration> element.

Important Note: The Java merger will respect the settings of the <indentation>configuration if the "lexicalPreserving" value is "false".

Optional Attributes

Attribute Description
lexicalPreserving This can be either "True" or "False". "False" is the default.
importSortStrategy This can be either "Eclipse", "IntelliJ", or "Default". The default value is "Eclipse".
mergeStrategy This can be either "Merge_Into_Existing" or "Merge_Into_New". The default is "Merge_Into_Existing".

Child Elements

None.

Example

This element specifies that we want to use the "IntelliJ" import sorting strategy:

<javaMergeConfiguration importSortStrategy="IntelliJ" />
